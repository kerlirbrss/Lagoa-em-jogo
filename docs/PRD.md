# PRD - Lagoa em Jogo

## 1. Visão Geral do Produto

### Nome

Lagoa em Jogo

### Slogan

"Feito por quem joga. Acompanhado por quem torce."

### Descrição

Lagoa em Jogo é uma plataforma digital voltada para o acompanhamento do futebol amador e comunitário de Lagoa de São Francisco - PI. O sistema centraliza informações sobre campeonatos, times, atletas, notícias, galerias fotográficas e estatísticas esportivas, promovendo maior visibilidade para atletas, organizadores, fotógrafos e torcedores locais.

---

## 2. Problema

Atualmente as informações sobre campeonatos, jogos, resultados, atletas e fotografias esportivas encontram-se dispersas em grupos de WhatsApp, redes sociais e aplicativos de terceiros, dificultando o acompanhamento organizado das competições locais.

---

## 3. Objetivo

Criar uma plataforma única para divulgação, acompanhamento e preservação da memória do futebol local.

# 19. Requisitos Funcionais (RF)

## RF01 - Visualizar Página Inicial

O sistema deve permitir que qualquer visitante visualize a página inicial contendo notícias, jogos, classificações e destaques.

## RF02 - Visualizar Campeonatos

O sistema deve permitir consultar informações completas dos campeonatos cadastrados.

## RF03 - Visualizar Classificação

O sistema deve exibir a classificação atualizada dos campeonatos.

## RF04 - Visualizar Calendário de Jogos

O sistema deve exibir datas, horários, campos e localizações das partidas.

## RF05 - Visualizar Perfil de Time

O sistema deve permitir consultar informações, elenco e estatísticas dos times.

## RF06 - Visualizar Perfil de Atleta

O sistema deve permitir consultar informações e estatísticas dos atletas.

## RF07 - Visualizar Notícias

O sistema deve permitir a leitura de notícias esportivas.

## RF08 - Visualizar Galerias

O sistema deve permitir visualizar galerias de fotos associadas a jogos, campeonatos e eventos.

## RF09 - Realizar Pesquisa

O sistema deve permitir pesquisar campeonatos, times, atletas e notícias.

## RF10 - Criar Conta

O sistema deve permitir o cadastro de novos usuários.

## RF11 - Realizar Login

O sistema deve permitir a autenticação de usuários cadastrados.

## RF12 - Comentar Notícias

O sistema deve permitir que usuários autenticados publiquem comentários em notícias.

## RF13 - Participar de Palpites

O sistema deve permitir que usuários autenticados realizem palpites em partidas futuras.

## RF14 - Favoritar Times

O sistema deve permitir favoritar times.

## RF15 - Favoritar Campeonatos

O sistema deve permitir favoritar campeonatos.

## RF16 - Configurar Notificações

O sistema deve permitir que usuários configurem notificações personalizadas.

## RF17 - Cadastrar Campeonatos

O sistema deve permitir que organizadores criem campeonatos.

## RF18 - Editar Campeonatos

O sistema deve permitir que organizadores editem campeonatos sob sua responsabilidade.

## RF19 - Cadastrar Jogos

O sistema deve permitir o cadastro de partidas.

## RF20 - Atualizar Resultados

O sistema deve permitir a atualização dos resultados das partidas.

## RF21 - Atualizar Classificação

O sistema deve atualizar automaticamente a classificação após a inserção de resultados.

## RF22 - Registrar Premiações

O sistema deve permitir registrar premiações coletivas e individuais.

## RF23 - Publicar Notícias

O sistema deve permitir a publicação de notícias por administradores e organizadores autorizados.

## RF24 - Publicar Galerias

O sistema deve permitir que fotógrafos parceiros publiquem galerias.

## RF25 - Vincular Fotos

O sistema deve permitir vincular fotografias a jogos e campeonatos.

## RF26 - Gerenciar Usuários

O sistema deve permitir que administradores gerenciem usuários cadastrados.

## RF27 - Gerenciar Fotógrafos Parceiros

O sistema deve permitir que administradores gerenciem fotógrafos parceiros.

## RF28 - Moderar Comentários

O sistema deve permitir que administradores removam comentários inadequados.

## RF29 - Personalizar Página Inicial

O sistema deve exibir conteúdo personalizado com base nos favoritos do usuário.

## RF30 - Exibir Informações dos Jogos

O sistema deve exibir campeonato, fase, rodada, horário, campo, localização e placar das partidas.

# 20. Requisitos Não Funcionais (RNF)

## RNF01 - Responsividade

O sistema deve funcionar adequadamente em smartphones, tablets e computadores.

## RNF02 - Mobile First

A experiência principal deve ser otimizada para dispositivos móveis.

## RNF03 - Usabilidade

A interface deve ser simples e intuitiva para usuários de diferentes faixas etárias.

## RNF04 - Acessibilidade

O sistema deve utilizar fontes legíveis, contraste adequado e navegação clara.

## RNF05 - Desempenho

As páginas devem carregar rapidamente mesmo em conexões móveis.

## RNF06 - Disponibilidade

O sistema deve estar disponível continuamente, salvo períodos de manutenção.

## RNF07 - Segurança

Somente usuários autenticados poderão acessar funcionalidades restritas.

## RNF08 - Controle de Permissões

O sistema deve respeitar os níveis de acesso definidos para cada perfil de usuário.

## RNF09 - Integridade dos Dados

As informações cadastradas devem permanecer consistentes e protegidas contra alterações indevidas.

## RNF10 - Escalabilidade

A arquitetura deve permitir a inclusão de novas modalidades esportivas no futuro.

## RNF11 - Manutenibilidade

O código deve ser organizado e documentado para facilitar futuras alterações.

## RNF12 - Compatibilidade

O sistema deve funcionar nos principais navegadores modernos.

## RNF13 - Confiabilidade

As informações exibidas devem refletir fielmente os dados cadastrados pelos responsáveis.

## RNF14 - Identidade Visual

Toda a plataforma deve seguir a identidade visual oficial do Lagoa em Jogo, utilizando as cores azul, verde e branco.

## RNF15 - Experiência do Usuário

A navegação deve exigir o menor número possível de cliques para acessar informações importantes.
